# Аналитика квартир

Простой статический сайт для GitHub Pages. Сайт читает CSV-файлы из папки `data`, сравнивает выгрузки по датам и показывает аналитику по квартирам.

## Формат CSV

Разделитель: `;`

Рекомендуемый порядок колонок:

```csv
project;link;area;price_with_discount;price_without_discount;flat_id;floor;building_name;section_number;section_floors_count
```

Описание полей:

- `project` — название проекта / ЖК
- `link` — ссылка на квартиру
- `area` — площадь
- `price_with_discount` — актуальная цена на сайте
- `price_without_discount` — цена без акции / полная цена
- `flat_id` — ID квартиры
- `floor` — этаж квартиры
- `building_name` — дом, например `Дом 10`
- `section_number` — номер секции
- `section_floors_count` — количество этажей в секции

Старые CSV без новых колонок тоже откроются, просто новые поля будут пустыми.

## Важная логика

Цены берутся из выбранной выгрузки.

Постоянные данные квартиры — `project`, `link`, `area`, `floor`, `building_name`, `section_number`, `section_floors_count` — подтягиваются из самой свежей строки по `flat_id`. Это нужно, чтобы старые выгрузки без новых полей могли нормально отображаться после добавления новых CSV.

## Как добавить новую выгрузку

1. Положи новый CSV в папку `data`, например:

```text
27.06.2026.csv
```

2. Добавь файл в `data/files.json`:

```json
[
  "13.06.2026.csv",
  "27.06.2026.csv"
]
```

3. Обнови сайт.

## Как проверить локально

В папке проекта запусти:

```bash
python3 -m http.server 8000
```

Потом открой:

```text
http://localhost:8000
```

На Windows команда может быть:

```bash
python -m http.server 8000
```

## Как залить на GitHub Pages

1. Создай репозиторий на GitHub.
2. Загрузи туда все файлы проекта.
3. Открой `Settings` → `Pages`.
4. В `Build and deployment` выбери:
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/root`
5. Сохрани настройки.

Через некоторое время GitHub даст ссылку на сайт.


## Сортировка таблиц

В таблицах можно нажимать на названия столбцов. Первый клик сортирует по возрастанию, второй — по убыванию. Активная сортировка отмечается стрелкой рядом с названием столбца.
